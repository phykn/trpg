### 흥미로운 관찰
- `최후의 호흡` 메시지가 두 턴 연속 출력됨 (소생 0/3 상태에서 1번 사용 후, 다음 사망에서도 동일 라벨 + HP 0→1 부활). 추정: `engines/combat.py:426`의 `revive_coins > 0` 게이트는 첫 0 도달까지만 발동하지만, 후속 dying 상태에서 death_save가 안정화(stabilize at hp=auto_revive_hp=1)되며 `format_combat_revived(coins_after=0)`가 한 번 더 출력됨. 결과적으로 "최후의 호흡 → 다음 사망 → 또 한 번의 최후의 호흡 → 그 다음 사망에서 GameOver" 시퀀스. UX상 두 번째 "최후의 호흡"이 첫 번째와 동일 라벨이라 플레이어 입장에선 "이거 무한 부활인가?" 헷갈릴 여지 있음. 차기 점검 후보.

### 신규 발견 결함 — narration 위치 모순 + 텍스트 중복
**1. 위치 모순 + 퀘스트 완료 흐름 미진행 (전체 흐름 재현)**
- 재현 시퀀스 (정공 검증):
  1. 광장에서 에드릭 퀘스트 수락 → `퀘스트 시작: 촌장의 부탁` 정상 카드
  2. 입력 `곧장 산문 길로 향해 약탈자를 찾는다` → narrate가 산문으로의 이동을 묘사
  3. 입력 `또 다른 약탈자를 발견하고 단검으로 정면 공격한다` → 그러나 전투 narration은 `이스나르 광장의 짙은 안개 속에서 ... 단검을 내리꽂습니다`로 광장에서 일어남으로 묘사. 처치 NPC도 `골목길 약탈자` (광장 entity)
  4. 입력 `광장으로 돌아가 촌장 에드릭에게 약탈자 처치를 보고한다` → narrate에선 에드릭이 보고를 자연스럽게 수락 (`정말 고맙소`)
- 결과: 퀘스트 패널은 보고 후에도 `쉬움 · 0/1` 그대로. ❌ 퀘스트 완료 카드 없음, ❌ GOLD 40 / EXP 80 보상 미지급 (HUD: 금화 0, EXP 8/100 변동 없음), ❌ 호감도만 `+5` 추가됨.
- 근본 원인 추정: quest engine은 `kill_target_of` 엣지 매칭(`server/src/engines/quest.py:368`)으로 진행하므로, scenario seed의 `흑탑 약탈자` ID가 아닌 동적 spawn된 광장/골목길 raider는 카운트 안 됨. 그리고 LLM이 `산문 길로 이동` 입력을 받았음에도 전투 narrate가 광장에서 일어났다고 묘사 — move action 분류 누락 또는 location pin 미적용 의심.
- 차기 세션 검토: (a) move action 분류가 안 됐는지, (b) location pin이 narrate context에 안 들어갔는지, (c) summon이 wrong location에서 spawn했는지, (d) quest engine의 kill_target 매칭을 동적 spawn entity까지 확장할지 vs 시나리오 seed에 quest target raider를 산문 길에 미리 배치할지 정책 결정.

- 차기 세션 검토: SSE `narrative_delta` 이벤트 시퀀스 + 클라이언트 본문 카드 어셈블리.
