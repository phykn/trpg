# Cycle 4: Authoritative Reference Binding

## Claim

A reference should remain bounded `ContinuityContext` while it only helps interpret recent language; it should become `GroundedAffordanceState` when it must survive the bounded context window to ground future inputs reliably.

## Evidence

- `server/src/game/runtime/flow/input.py:65-72` sets classifier limits, including `recent_dialogue=5`.
- `server/src/game/runtime/flow/input.py:722-727` persists `active_subject_id` when a subject is produced.
- `server/src/llm/context/classify_view.py:92-97` exposes `last_npc`, `last_target`, `last_item`, and `recent_dialogue` under `references`.
- `server/src/llm/context/classify_view.py:364-375` derives last entity references from `runtime.progress.active_subject_id`.
- `server/tests/llm/context/test_classify_view.py:160-185` verifies recent dialogue can be limited to the last two entries.
- `server/tests/llm/context/test_classify_view.py:191-214` verifies grounding view preserves ids for player, NPC, connection, inventory, and skills.
- `server/src/db/graph/local_fs.py:125-139` loads only tail history and dialogue entries by configured limits.
- `server/src/game/rules/config.py:40-43` sets `turn_log_size=50` and `recent_dialogue_turns=10`.

## Objection

The claim is too window-centered.

The important question is not whether a reference must survive beyond a bounded context window. The important question is whether the reference is an authoritative binding for grounded action.

Some references affect the next grounded action immediately, even before any context window expires. "Talk to them again" or "attack that one" may keep the available verb set unchanged while changing the target id. That means reference binding is part of grounded action, not merely long-lived context.

Additional evidence:

- `server/src/locale/prompts/classify/prompt.ko.md:227-234` requires all ids to exist in context, says ambiguous matches should `pass`, and restricts `context.references` to pronoun/reference resolution.
- `server/tests/game/runtime/test_graph_input.py:847-860` verifies targetless speak resolves to a nearby living NPC and persists `active_subject_id == "goblin_01"`.

## Revision

Replace the survival-window test with an authority test.

Revised rule:

```text
Reference state is authoritative binding for grounded action, not long-lived context.
```

A reference remains `ContinuityContext` when it only helps interpret local wording.

A reference becomes `GroundedAffordanceState` when removing it changes the grounded interpretation of an input:

- target id
- action binding
- pending resolution
- valid transition

## Why It Is Stronger

The revision removes a misleading time-based criterion.

It explains why `recent_dialogue` can stay bounded while `active_subject_id` is persisted: recent dialogue helps local interpretation, but active subject acts as an authoritative binding for later grounded inputs.
