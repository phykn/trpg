# Theory Cycle 13

## Claim

LLM-TRPG play remains possible when two minimum conditions are preserved:

```text
PlayableContinuity =
TransitionValidity + MeaningContinuity
```

- `TransitionValidity`: the next move is possible under the game's constraints.
- `MeaningContinuity`: the next moment remains connected to prior fiction, obligations, and player intent.

## Evidence

- A tabletop game can continue without full memory if the next ruling is valid and the fiction still makes sense.
- A computer RPG can continue without simulating every world detail if the accepted next state follows rules and preserves quest/story continuity.
- Interactive fiction can continue with partial parser knowledge if the next accepted command remains valid and intelligible.
- LLM narration does not need full context, but it must avoid impossible transitions and disconnected narration.

## Objection

This mistakes coherent continuation for playability.

A system can preserve valid transitions and meaningful narration while still destroying play:

- every player action is redirected,
- all choices secretly lead to the same state,
- available actions are not legible,
- the narrator overwrites the player's declared intent.

So the original formula is not sufficient. A coherent LLM novel generator could satisfy it while failing to be a playable TRPG.

It is also too strict at the raw-output level. Play can survive an invalid or discontinuous response if the system repairs it before it becomes accepted fiction.

## Revision

Revise the top-level theory:

```text
PlayableContinuity =
TransitionValidity + MeaningContinuity + AgencyContinuity
```

This applies to accepted game state, not every raw model output.

- `TransitionValidity`: the accepted transition is allowed by rules, constraints, genre contract, or agreed exception.
- `MeaningContinuity`: the accepted state remains connected to prior fiction, obligations, and stakes.
- `AgencyContinuity`: the player's declared intent, available affordances, and expectation of consequence remain legible enough to support the next meaningful choice.
- Repair can happen before acceptance. Broken raw output does not break play if it is corrected before becoming game state.

## Why it is stronger

The revision separates playable game continuity from mere coherent narration. It also keeps the original minimum-context insight: full memory and perfect simulation are unnecessary, but valid transition, meaningful fiction, and player agency must survive.

## Next question

What failures belong to `MeaningContinuity`, and what failures require the separate condition `AgencyContinuity`?
