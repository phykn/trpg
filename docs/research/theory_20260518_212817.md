# Theory Cycle 11

## Claim

Classifier context counts as `TransitionState` when content-backed facts shape the LLM's proposed action.

## Evidence

- `server/src/llm/context/classify_view.py` builds classifier context with identity, affordances, references, and budget.
- `classify_context_to_grounding_view` reduces classifier context to a grounding view.
- `server/src/llm/calls/classify/grounding.py` validates proposed actions against ids from that grounding view.
- `server/src/locale/prompts/classify/prompt.ko.md` says the LLM chooses intent and candidate ids, but does not create final Action JSON, success, failure, difficulty, damage, or reward.
- `server/src/game/runtime/flow/input.py` sends the classified action through roll, narrative, or runtime request paths, and runtime rejection is still possible.

## Objection

Classifier context is too broad to equal `TransitionState`.

It mixes:

- identity and affordance facts projected from transition authority
- reference and recent-dialogue aids from continuity
- current player input
- budget metadata about truncated candidates

The LLM can propose an action from this context, but proposal is not final transition authority. Grounding can reject unknown ids and local non-affordances, while dispatch and runtime engines still decide deeper validity.

## Revision

Add:

```text
ActionProposalContext =
TransitionStateProjection + ContinuityContextProjection + CurrentInput + BudgetMetadata
```

`ActionProposalContext` is not a third condition. It is a bounded prompt envelope for proposing actions.

`GroundingView` is the `TransitionState`-derived subset used to constrain proposed actions to known ids and local affordances.

`GroundingView` grounds proposals, but does not decide final transition validity. Dispatch and runtime engines still enforce deeper transition rules.

## Why it is stronger

This keeps `TransitionState` authoritative without pretending every prompt field is state. It also explains why bounded context can help choose actions while still being insufficient to authorize them.
