# Cycle 3: Grounded Action Space

## Claim

Information should move from `ContinuityContext` into `AffordanceState` iff removing it changes valid next actions or unresolved process handling.

## Evidence

- `server/src/game/runtime/flow/input.py:402-410` blocks free input when `pending_confirmation` or `pending_roll` exists.
- `server/src/game/runtime/flow/confirmation.py:217-230` and `server/src/game/runtime/flow/confirmation.py:285-298` require matching pending confirmation and clear it after resolution.
- `server/src/game/runtime/flow/roll.py:105-123` creates pending roll state; `server/src/game/runtime/flow/roll.py:350-379` requires the pending roll id, loads the pending action, resolves it, and clears the pending roll.
- `server/src/llm/context/classify_view.py:80-97` separates `affordances` from `references`.
- `server/src/llm/context/classify_view.py:378-385` includes bounded recent dialogue summaries in classifier context.
- `server/src/game/runtime/narration/memory_context.py:88-114` selects related memory by related ids and importance.
- `server/src/game/runtime/narration/memory_context.py:121-135` derives related ids from current location, active subject, active quest, target, and action fields.
- `server/src/game/runtime/narration/context.py:320-328` includes public knowledge in narration payload.
- `server/src/game/runtime/narration/context.py:496-514` derives public knowledge from graph `has_knowledge` edges and visibility.

## Objection

The claim is too narrow if `valid next actions` means only an action list.

Removing context can leave the same verbs available while changing what the next input means. For example, "talk to them again", "use that", or "go to the place from before" may keep `can_speak_to`, `can_use`, and `can_move_to` unchanged, but change the grounded target or reference binding.

The code supports this objection because classifier context separates:

- `affordances`: what action classes are possible
- `references`: last npc, last target, last item, and recent dialogue

So the decisive object is not a bare action set. It is the grounded action space.

## Revision

Replace `valid next actions` with `grounded action space`.

Revised formula:

```text
MinConditions = GroundedAffordanceState + ContinuityContext
```

`GroundedAffordanceState` preserves the minimum authoritative data needed to determine:

- valid verbs
- valid targets
- reference bindings
- required pending resolution

Information moves from `ContinuityContext` into `GroundedAffordanceState` iff removing it changes the grounded action space or unresolved process handling.

## Why It Is Stronger

The revision keeps the core claim but fixes the weak definition.

It explains why references and recent dialogue can matter even when they do not add new verbs. It also keeps a boundary: context can remain bounded while it helps interpretation, but must become state when losing it changes executable grounding.
