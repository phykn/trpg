# Theory Cycle 10

## Claim

`RuntimeContent` facts belong in `TransitionState` when they affect transitions, and otherwise belong outside `MinConditions` or in `ContinuityContext`.

## Evidence

- `server/src/game/domain/progress.py` persists `runtime_content`.
- `server/src/game/runtime/load.py` merges scenario content with `progress.runtime_content`.
- `server/src/game/domain/content.py` defines `node_value`: graph node properties are checked first, then content records are used as fallback.
- `server/src/game/runtime/narration/context.py` uses content-backed dialogue style, MBTI, public knowledge, role, item kind, and price for narration payloads.
- `server/src/game/runtime/narration/combat_view.py` uses content-backed effect and status descriptions for combat narration views.
- `server/src/game/engines/graph/transfer.py` reads trade price directly from item graph properties.
- `server/src/game/engines/graph/item_use.py` reads effect mechanics from effect node properties.
- `server/src/game/seed/graph_seed.py` creates graph nodes and edges for content-backed records such as knowledge, dialogue style, MBTI, effects, statuses, actions, and quests.

## Objection

The split is too direct. `RuntimeContent` is not itself a condition category. It is an authority source.

Putting all content-backed facts into `TransitionState` is too broad. Excluding them as static schema is too narrow. The same merged content source can feed labels, narration constraints, public knowledge wording, prompt payloads, and sometimes runtime-created facts.

Whether a content-backed fact enters `TransitionState` depends on the consuming runtime path.

## Revision

Keep:

```text
MinConditions = TransitionState + ContinuityContext
```

Add:

```text
ContentAuthority = ScenarioContent + RuntimeContentOverlay
```

`ContentAuthority` is a source of authoritative facts, not a third condition.

A content-backed fact enters `TransitionState` only when transition, process, validation, or durable truth code consumes it as authority.

A content-backed fact enters `ContinuityContext` or presentation payloads when it only constrains labels, narration, dialogue style, public knowledge wording, or prompt-facing description.

## Why it is stronger

This keeps the minimal condition formula intact while making authority sources explicit. It prevents both mistakes: treating all content as transition state, and treating content as harmless static text.
