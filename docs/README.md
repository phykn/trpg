# TRPG docs

이 문서는 코드 사용법이 아니라 설계 문서의 입구다. 현재 계획의 기준은 `docs/plan.md`다.

## 핵심 생각

이 게임은 graph 기반 TRPG다. 캐릭터, 아이템, 장소, 퀘스트, 기술은 graph node이고, 위치, 소유, 장착, 이동 연결, 퀘스트 목표와 보상은 edge다.

LLM은 플레이어 말뜻을 고르고 결과를 이야기로 쓴다. engine은 게임에서 실제로 일어난 일을 정하고 graph를 바꾼다.

## 먼저 읽을 문서

| 파일 | 내용 |
|---|---|
| `docs/plan.md` | 프로젝트 전체 설명, 재작성 판단, LLM 파이프라인, runtime 계약 |

## 보조 문서

| 폴더 | 내용 |
|---|---|
| `docs/ui/` | client 제품 스타일과 UI 세션 기록 |
| `docs/supabase/` | graph 저장 테이블 SQL |
| `docs/superpowers/` | 작업 계획과 설계 기록 |

## 문서 쓰는 규칙

- 같은 규칙은 한 곳에만 자세히 쓴다.
- 어려운 단어는 처음 나올 때 쉬운 뜻을 붙인다.
- LLM이 해도 되는 일과 하면 안 되는 일을 나눠 쓴다.
- 한국어 문체 예시는 `ko` locale의 예시다.
